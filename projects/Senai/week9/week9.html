Week 9 – Senai Web Development Course

Overview

This page contains all the React-based development activities completed in Week 9 of the Senai Web Development course. The focus of this week is on creating a functional daily task manager and implementing core features such as state management, conditional rendering, persistence, and user interaction.

Tasks

✅ Task 1: Task Manager – Add New Task

Description: We are building a daily task manager. Each task must include:

  • A description field;
  • A time period selector (Morning / Afternoon / Evening);
  • A completed status (checkbox).

Create a component that allows users to add a new task by filling in the description, selecting the period, and clicking an "Add" button. The new task should be added to the task array in the main component.

✅ Task 2: Task Manager – Display Tasks

Description: Display the list of tasks in three separate columns based on their period (Morning, Afternoon, Evening). This should appear below the task addition form.

Also, show the total number of completed tasks.

✅ Task 3: Task Manager – Mark as Completed

Description: Each task item should include a checkbox to mark it as completed. When a task is marked as completed, its description should appear in light gray and be strikethrough.

✅ Task 4: Task Manager – Progress Bar

Description: Create a component that displays the percentage of tasks completed. This component should receive at least two props: the total number of tasks and the number of completed tasks.

const progress = (completedTasks / totalTasks) * 100

✅ Task 5: Task Manager – Data Persistence

Description: Implement data persistence using localStorage. Any task added or updated should be stored. When the app loads, check localStorage for existing tasks and load them into state.

Tip: Use useEffect to synchronize the task list with localStorage.

✅ Task 6: Stopwatch Challenge

Description: Deliver a stopwatch project that was developed in class during Week 9 (Session 2). The activity was assigned during the session and had a fixed time for development.

Submission must be made via GitHub link according to instructions given during class.
